# 1. Checksum, Hash

## 1.1 Checksum

- 데이터의 오류 여부 확인 방법으로 널리 사용
- 일정 자릿수를 정하고 범위를 넘는 자리 올림은 버려서 자릿수를 유지

- 자리올림 1을 버리면 결과는 6

## 1.2 Hash

- 단방향성
    - 결과값을 가지고 입력값을 찾을 수 없다.
- 입력 값의 크기와 상관 없이 결과 값의 길이(혹은 크기)가 일정
- 데이터 무결성 확보와 관련해 IT기술 전반에서 사용된다.
- 대표적인 Hash 알고리즘
    - SHA-128, 256, 384, 512

# 2. 대칭키, 비대칭키

## 대칭키

- 키 하나로 암호화/복호화를 모두 수행하
는 방식.
- 비대칭키 방식에 비해 효율적이다.
- DES, 3DES, SEED-128, ARIA, AES-128, AES-256 알고리즘이 유명
하다.

```java
0 1 0 0 0 0 0 1
---------------

---------------

```
