### 1. 개념: 가벼운 객체 만들기
* **정의:** 많은 수의 객체를 생성할 때, 공통된 부분(Intrinsic)을 공유하여 메모리 사용량을 줄이는 패턴임.
* **핵심:**
    * **Intrinsic (내적 속성):** 변하지 않고 공유되는 데이터 (예: 폰트 모양, 글자 색).
    * **Extrinsic (외적 속성):** 객체마다 달라지는 데이터 (예: 글자 위치).
* **목적:** 인스턴스 생성을 줄여 **메모리 낭비(OOM)를 방지**하는 것임.

### 2. 구현 방식
* **불변성:** 공유되는 객체(Flyweight)는 반드시 **불변(Immutable)**이어야 함.
* **팩토리 활용:** 클라이언트는 `new`로 객체를 직접 만들지 않고, **팩토리(Factory)**를 통해 이미 만들어진 객체를 받아옴 (캐싱).

### 3. 장단점
* **장점:** 엄청난 수의 객체를 다룰 때 **메모리 사용량을 획기적으로 줄일 수 있음.**
* **단점:** 코드 구조가 복잡해짐. (속성을 분리하고 팩토리를 경유해야 하므로).

### 4. 실무 활용 (Java)
* **`Integer.valueOf(int)`:**
    * `-128`부터 `127`까지의 숫자는 미리 생성된 객체(Cache)를 재사용함.
    * 그래서 `==` 비교 시 `true`가 나오지만, 범위를 벗어나면 `new`로 생성되어 `false`가 나옴.
* **`String Constant Pool`:** 문자열 리터럴(`"hello"`)을 재사용하는 것도 같은 원리임.

### 요약
* 플라이웨이트 패턴은 **"자주 쓰는 객체를 캐싱해서 돌려쓰기(공유)함으로써 메모리를 아끼는 기술"**임.
