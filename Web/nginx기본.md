Q) Nginx의 정의
A) 웹 서버, 리버스 프록시, 로드 밸런서, HTTP 캐시 등으로 사용되는 고성능 이벤트 기반 소프트웨어.


Q) Nginx가 해결하려 한 핵심 문제
A) C10K 문제 (1만 개 이상의 동시 연결을 효율적으로 처리하는 문제)

Q) Nginx가 C10K 문제를 해결한 핵심 아키텍처
A) 이벤트 기반 구조 (Event-Driven) / 비동기 논블로킹 (Async Non-Blocking)


Q) 전통적인 웹 서버(아파치)의 연결 방식
A) 요청마다 프로세스 또는 스레드를 할당하는 방식 (Prefork, Worker).

Q) 아파치 방식의 단점 (C10K 발생 원인)
A) 동시 접속자 증가 시 메모리 사용량이 급증하고, 잦은 컨텍스트 스위칭으로 CPU 오버헤드 발생.


Q) Nginx 워커 프로세스 (Worker Process)
A) 적은 수(보통 CPU 코어 수)의 프로세스가 이벤트 큐를 통해 여러 연결을 동시에 효율적으로 처리.


Q) Nginx 이벤트 기반 구조의 최대 장점
A) 매우 적은 메모리 사용으로 수많은 동시 연결을 효율적으로 처리 가능 (높은 성능).


Q) Nginx의 웹 서버(Web Server) 기능
A) Nginx는 웹 서버(Web Server) 역할을 할 수 있다. 풀어서 설명하자면 Nginx가 사용자의 요청이 들어올 때마다 파일(HTML, CSS, JS, 이미
지 등)을 제공할 수 있는 기능을 가지고 있다.
