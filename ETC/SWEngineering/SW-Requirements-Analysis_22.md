# 요구사항과 테스트의 관계 (V-Model)

## 1\. 요구사항과 테스트의 연결

  * **핵심:** 요구사항 분석과 명세를 제대로 했다면, 사실상 \*\*테스트 케이스(Test Case)\*\*까지 도출된 것과 다름없음.
  * **검증 방법:** 작성된 요구사항을 다른 사람(QA, 개발자 등)에게 보여주고 \*\*"이걸 보고 테스트할 수 있겠니?"\*\*라고 물었을 때, "Yes"가 나오면 잘 된 요구사항이고, "No"가 나오면 잘못된 것임.

-----

## 2\. V-Model (V-모델)

소프트웨어 개발 생명주기(SDLC)에서 개발 단계와 테스트 단계가 어떻게 매핑되는지를 보여주는 모델.

[Image of V-Model in software testing]

| 왼쪽 (개발/모델링) | 오른쪽 (테스트/검증) | 관계 |
| :--- | :--- | :--- |
| **요구사항 분석**<br>(Requirements Analysis) | **인수 테스트**<br>(Acceptance Test) | 사용자의 요구사항(관심사항)이 충족되었는지 검증. |
| **시스템 설계**<br>(System Design) | **시스템 테스트**<br>(System Test) | 전체 시스템의 기능/비기능 요구사항 만족 여부 검증. |
| **아키텍처 설계**<br>(Architecture Design) | **통합 테스트**<br>(Integration Test) | 모듈 간 인터페이스 및 상호작용 검증. |
| **모듈 설계**<br>(Module Design) | **단위 테스트**<br>(Unit Test) | 개별 모듈(함수/클래스)의 기능 검증. |
| **구현** (Implementation) | - | 코딩 단계. |

-----

## 3\. Testability (테스트 용이성)

  * **정의:** 요구사항이 테스트 가능한 형태로 작성되었는가?
  * **기능 요구사항:** `Input` 대비 `Output` 변화가 명확하면 테스트가 매우 쉬움 (Input 넣고 Output 확인 $\rightarrow$ Pass/Fail).
  * **비기능 요구사항:** '빠르게'가 아니라 '3초 이내'라고 수치화되어야 테스트 가능함 (Stimulus $\rightarrow$ Response 측정).
  * **결론:** **Testability가 확보된 요구사항 = 잘 분석된 요구사항.**

## 4\. 소프트웨어 테스트의 종류

### ① 단계별 분류 (V-Model 기반)

1.  **단위 테스트 (Unit Test):** 개발자 관점. 모듈(함수, 클래스, 파일) 단위의 기능 검증.
2.  **통합 테스트 (Integration Test):** 모듈들을 결합했을 때 인터페이스가 잘 동작하는지 검증.
3.  **시스템 테스트 (System Test):** 전체 시스템이 기능/비기능 요구사항을 만족하는지 검증.
4.  **인수 테스트 (Acceptance Test):** 사용자(User) 관점. 실제 환경에서 사용자가 만족하는지 검증 (알파/베타 테스트 포함).

### ② 기법별 분류

  * **블랙박스 테스트 (Black-box):** 내부 로직을 보지 않고 Input/Output만 확인 (주로 기능 테스트).
  * **화이트박스 테스트 (White-box):** 내부 소스 코드 로직과 경로를 보면서 테스트 (Code Coverage 측정 등).

### ③ 실행 시점별 분류

  * **정적 테스트 (Static):** 코드를 실행하지 않고 컴파일 단계나 코드 리뷰를 통해 버그를 찾음 (정적 분석 도구 활용).
  * **동적 테스트 (Dynamic):** 실제 코드를 실행하면서 버그를 찾음.

-----

## 5\. 자동화 테스트 (Test Automation)의 중요성

  * 현대 소프트웨어는 \*\*상호운용성(Interoperability)\*\*이 중요하고, OS 업그레이드나 오픈소스 교체 등이 빈번하게 일어남.
  * 매번 사람이 수동으로 테스트하는 것은 불가능하므로, **단위/통합/시스템 테스트의 자동화**와 \*\*회귀 테스트(Regression Test)\*\*가 필수적임.
