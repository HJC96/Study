### **1. 압도적인 시장 점유율과 생태계**
* **업계 표준:** 페이스북(Meta), 인스타그램, 넷플릭스 등 글로벌 기업들이 사용하며, Vue나 Angular 등 타 도구 대비 가장 높은 점유율을 차지하고 있습니다.
* **이점:** 채용 수요가 많아 취업에 유리하며, 방대한 학습 자료와 커뮤니티가 형성되어 있어 배우기 수월합니다.


### **2. 뛰어난 반응성 (Reactivity)**
과거 웹페이지는 작은 변화(예: 좋아요 버튼 클릭)에도 전체 페이지를 새로고침해야 했습니다. 하지만 리액트는 **AJAX** 기술 등을 활용하여 필요한 데이터만 서버에서 받아 부분적으로 업데이트합니다.
* **결과:** 화면 깜빡임 없이 쾌적한 사용자 경험(UX)을 제공하며, 무한 스크롤 같은 기능을 자연스럽게 구현할 수 있습니다.


### **3. 컴포넌트 기반의 재사용성 (Reusability)**
기존 HTML 방식은 중복 코드가 많고 수정이 번거로웠습니다. 리액트는 UI 요소를 **'컴포넌트(Component)'**라는 독립적인 조각으로 나눕니다.
* **이점:** 레고 블록처럼 필요한 곳에 조립하여 재사용할 수 있어, 코드 중복을 줄이고 유지보수 효율을 극대화합니다.


### **4. 직관적인 문법 (JSX)**
순수 자바스크립트로 DOM을 조작하려면 코드가 길고 복잡해지기 쉽습니다. 리액트는 JavaScript와 HTML을 합친 듯한 **JSX(JavaScript XML)** 문법을 사용합니다.
* **이점:** 코드의 가독성이 높고, UI 구조와 기능을 한눈에 파악하기 쉽습니다.

### **5. 가상 돔 (Virtual DOM)을 통한 성능 최적화**
브라우저가 실제 DOM을 조작하는 것은 연산 비용이 많이 듭니다(비유: 주차장에서 차를 실제로 계속 옮기는 것). 리액트는 메모리 상에 **가상 돔(Virtual DOM)**을 만들어 미리 계산합니다.
* **작동 원리:** 가상 돔에서 변경 사항을 먼저 시뮬레이션(비유: 모형 차로 배치 연습)한 뒤, **실제 변경이 필요한 부분만** 골라 실제 DOM에 반영하여 웹페이지 성능을 높입니다.
