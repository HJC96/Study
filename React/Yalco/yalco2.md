### 1. DOM (Document Object Model, 문서 객체 모델)
정의: HTML 문서(텍스트 파일)를 브라우저가 이해하고 조작할 수 있도록 만든 '객체(Object)' 형태의 실체입니다.
비유:
     HTML: 로봇의 설계도 (단순한 문서)
     브라우저: 설계도를 보고 로봇을 만드는 공장
     DOM: 공장에서 만들어진 실제로 움직이는 로봇
역할: 자바스크립트가 웹페이지의 요소들을 잡고 흔들 수 있도록 만들어진 '손잡이(인터페이스)' 역할을 합니다. 이를 통해 정적인 문서에 생명을 불어넣어 동적인 기능을 수행하게 합니다.

### 2. CSSOM (CSS Object Model, CSS 객체 모델)
정의: DOM과 마찬가지로, CSS 파일이나 스타일 태그의 내용들을 브라우저가 이해할 수 있도록 만든 트리 구조의 객체 모델입니다.
특징: HTML의 DOM 트리와 별개로, 스타일 규칙들이 트리 형태로 구성됩니다.

### 3. 브라우저의 렌더링 과정 (화면이 그려지는 원리)
브라우저는 다음 과정을 거쳐 우리가 보는 화면을 만들어냅니다.
1. DOM + CSSOM 결합: DOM 트리와 CSSOM 트리를 합쳐 렌더 트리(Render Tree)를 만듭니다. 참고: 화면에 보이지 않는 요소(예: `display: none`)는 렌더 트리에서 제외됩니다.
2. 레이아웃(Layout): 각 요소가 화면의 어느 위치에 어떤 크기로 놓여야 할지 계산합니다.
3. 페인트(Paint): 계산된 위치에 실제로 색을 칠하고 내용을 그려 최종 화면을 보여줍니다.

### 4. 자바스크립트와의 관계 (Web API)
오해와 진실: DOM과 CSSOM은 자바스크립트 언어 자체가 아니라, 브라우저가 제공하는 Web API입니다.
상호작용: 자바스크립트는 이 Web API를 통해 DOM에 접근하여, 이미 로딩된 웹페이지의 내용(텍스트, 스타일, 요소 추가/삭제 등)을 동적으로 변경할 수 있습니다.
실무 팁: 스타일을 변경할 때 CSSOM을 직접 조작하기보다는, 주로 DOM을 통해 클래스를 변경하거나 인라인 스타일을 수정하는 방식을 사용합니다.
